# Правила для разработки 3D-игры (Three.js + Ammo.js)

**Стек проекта:** клиент — TypeScript ^5.3.3, Vite ^5, Three.js 0.183.1, Ammo.js (Bullet, WASM `ammo.wasm.js`), Miniplex ^2.0.0; сервер — Go 1.23, `github.com/coder/websocket` v1.8.14.

## Основные принципы
- Пиши решения, ориентированные на реальный игровой прод: читаемость, производительность, предсказуемость.
- Всегда разделяй рендер (`Three.js`) и физику (`Ammo.js`) по ответственности.
- Сначала предлагай минимально рабочее решение, затем улучшения.
- Избегай лишних аллокаций в игровом цикле и частых созданий `Vector3/Quaternion/Matrix4` в `update`.
- Для сложной математики и физики давай короткое объяснение "почему так".

## Код и архитектура
- Папка `client/` — код клиента (игрок, браузер); папка `server/` — код мультиплеерного сервера.
- Используй строгий `TypeScript` и понятные типы для ECS-компонентов и систем.
- Держи `deltaTime`-зависимую логику стабильной (кадр-независимое поведение).
- Проверяй корректность синхронизации трансформов между физикой и визуалом.
- Учитывай производительность: батчинг, frustum culling, переиспользование объектов, object pooling.
- Для правок в геймплее предлагай короткий план проверки (как воспроизвести, как убедиться, что фикс работает).

## Ограничение по попыткам
- Если решение не получается после 3 попыток, остановись и спроси пользователя о дальнейших действиях.
